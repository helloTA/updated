Ruby Iterators



Ruby iterators are nothing but a method which allows you to loop through all the members of a collection.  Ruby iterators works just like "for" or "While" but without any extra language keywords.

It executes a block of code again and again.
Ruby times iterator.
Times iterator used to execute a block of for n no of times.
You can explicitly mention how many no of times you want execute the
block.

Syntax:


  3.times do

  puts "Hello"

  end
Output:

  Hello

  Hello

  Hello
Example 2

  potatoes = 1

  3.times do |i|

  potatoes = i+1

  puts  "#{potatoes} potato"

  end

  puts potatoes + 1
Output:

  1 potato

  2 potato

  3 potato

Ruby Each Iterator.
Ruby Each iterator allows a collection to be executed each time.It returns all the elements in a collection. Ruby each interator will iterate through a list or an array, or a hash and loop through all the elements and return each element.
Syntax:


  collection.each do |variable|

  # Your code

  end

  numbers = [100,200,300,400]

  numbers.each do |n|

  puts "The number is #{n}"

  end

Output:


  The  number is 100

  The  number is 200

  The  number is 300

  The  number is 400

Ruby Collect Iterator
Array#each just takes each element and puts it into the block, then returns the original array. Array#collect takes each element and puts it into a new array that gets returned:
Look at the ruby collect iterator below.


  
[1, 2, 3].each { |x| x + 1 }    #=> [1, 2, 3]

  [1, 2, 3].collect { |x| x + 1 } #=> [2, 3, 4]

Example:


  arr = [1,2,3,4]

  arr.collect {|x| x*2}

  print x

Output:


   2,4,6,8

Ruby Select Iterator.
The way ruby select works is simple, it basically iterates through all the elements in your array and performs your logic on each one. If the logic returns TRUE, then it adds the item to a new array which it returns when the iteration is complete.
Here’s an example how ruby select works:


  
a = [1,2,3,4,5]

  a.select {|x| x > 2}

This will return the last three elements in the array.
Why? Because 3,4 and 5 are both greater than 2, which was the logic we placed in the block

Ruby reject iterator.
ruby reject iterator will perform the opposite operation of select. Logic which returns FALSE adds the item to the array that is returned. Here’s the same examples as before except we will swap select, with reject:
Here is an example of ruby reject:


  a = [1,2,3,4]

  a.reject {|n| n > 2}
In this example the return value is [1,2] because these elements return false when the condition is tested.

Ruby detect iterator:
Let’s just put it out there and say that other iterators could be used to select the correct value from the array, but I am going to show you how to use detect to round out our exploration of these iterators.

So let’s find the value 3 in our array using detect:
Here is an example of ruby detect iterator.


  a = [1,2,3,4]

  a.detect {|n| n == 3}
This returns 3. The value we were looking for. If the value had not been found, then the iterator returns nil.

When to use these iterators?
1.Use select or reject if you need to select or reject items based on a condition.
2. Use collect if you need to build an array of the results from logic in the block.
3. Use detect if you need to find an item in an array.
