Ruby Hashes



Similar to Arrays, Hashes are heterogenous. Which means a Hash can hold mixture of data types in the same Hash.

Hashes contains key-value pairs. In the key value pairs key means the referred value.

A Hash is a collection of key-value pairs. You retrieve or create a new entry in a Hash by referring to its key. Hashes are also called 'associative arrays', 'dictionary', 'HashMap' etc. in other languages

Just like array indexes we've keys in hashes.

if we want to use numeric indices that don't go in order from 0 to the end of the array? What if we don't want to use numbers as indices at all? We'll need a new array structure called a hash.

For example in 


  Array  a = ["apple", "banana",  "mango"]

  a[0] => apple

  a[1] => banana

  a[3] => mango

We can represent the same data using hashes more powerfully.
Example:       


  list = {"fruit 1" =>  "apple",

  "fruit 2" =>  "banana",

  "fruit  3" => "mango" 

  }                      
  puts list["fruit  1"] # apple

  puts list["fruit  2"] # banana

  puts list["fruit  3"] # mango
Creating ruby hashes:
Ruby hashes are more powerful data structure to represent data.
ruby hash can be declared using two curly braces {}. Here is an example of a Hash in Ruby:

Synatx :            


  hash = Hash.new

  hash = {

  key1 => value1,

  key2 => value2,

  key3 => value3

  }
  
  
student_ages = {

  "Jack" => 10,

  "Jill" => 12,

  "Bob" => 14

  }

Lets look at ruby hash example:


  restaurant_menu = {  "Idly" => 100, "Biryani" => 300, "Coffee"  => 200 }

  restaurant_menu.each  do | item, price |

  puts "#{item}: $#{price}"

Output:  


  Idly : 100

  Biryani : 300

  Coffee: 200
In the above example keys are "Idly", "Biryani", "Coffee"  and values are 100, 300, 200
