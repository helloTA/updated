Ruby modules and Mixins



Modules are the one of the key features in ruby programming language.
But what is Module in ruby?. Modules in ruby is way to organise group of or no of methods, classes,variables together in single module.

But what exactly ruby modules does?. Lets think about a situation where you want to write an application where you want to write more no of classes.You need to write more and more number of methods. How do you do that? How can you access each class?.

Lets say you are writing a program called Mobile Phones. You wrote 50 ruby classes in the program  and saved it in mobile_phones.rb file.Now you're writing Telephones program and you wrote 30 ruby classes. You need to write 10 more ruby classes but somehow  you came to know that Mobile Phones.rb file has the same set of 10 ruby classes which you need.

Now you've two options.
1. Write the 10 ruby classes again in Telephones.rb file
2. Use the Mobile Phones.rb classes.

If you're a smart programmer you would chose the second option to inherit
Mobile Phones.rb ruby classes. That is the power of ruby modules. Ruby modules does this job for you.

You can just use the code of other class using the modules without re-writing those classes again.

Take look at the below image. There is file called ruby-authentication.rb which has 50 ruby classes and 100 ruby methods which allows users to signup or authenticate from third party services
like facebook, twitter etc....

Lets say, if you're writing a ruby application to allow users to authenticate to your website from other services. Instead of writing authentication program for each service, here one program to authenticate from facebook, authenticate from twitter, authenticate from google+, authenticate from quora you can use a single authenticate program and implement the same for all the services with a single file with the help of ruby modules. That is why ruby modules are very powerful. Ruby modules
implements DRY feature. Which means Don't repeat yourself.
Ruby modules the powerful feature in doing such kind of tasks.

Ruby also provides name collision facility. Which means if you’ve 3 classes with same name ruby modules will help us to prevent this collision.
Ruby Modules Syntax:


  module Module_name

  
module Module_name  # Module name must start with Capital letter.

      def Module_name.method_one_name    # Creating a method with module

     def Module_name.method_two_name

     end

  end

Ruby Include Statement
Include statement in ruby is used to include one module in other modules.
If you want to work the different modules then you can simple mention the module which you want to incorporate in your program. Ruby modules can be used anywhere using ruby include statement.
Syntax for ruby include statement.


  include  module_name
Example Syntax:

  module  Fruits

  end

  module Vegetables

  include Fruits     #  when you include Fruits module then you have access to all methods, variables  and code

  end

Ruby Mixins

What is a Mixin in Ruby?. Mixins are used to implement multiple inheritance. Technically, ruby does not support multiple inheritance. So ruby mixins come in hand to implement the same feature in ruby using ruby mixins. Also Mixins provides fully controlled way to add functionality to our ruby class.

# Module One

 
  
module First

  def first_method

  puts "I am in  the first method of Module First"

  end

  def second_method

  puts "I am in  the second method of module First"

  end

  end

  Module Second

  def third_method

  puts "I am  in the third method of Module Two"

  end

  def fourth_method

  puts "I am in  the fourth method of module Two"

  end

  end

  # Now lets define a  class to play with.


class Testing

  # Now we want to  create two methods first_method, and second_method

  # Should we define  the same methods again in the class??? No1. Not.

  # Instead, we can just include the Module First to have  access to first_method, and second_method

  include First  # When you include First module you have  included the two method as well.

  t = Testing.new

  t.first_method

  t.second_method

  end
Output:
 
  I am in the first method of Module First

  I am in the second method of Module First
