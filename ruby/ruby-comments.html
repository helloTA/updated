Ruby comments



Comments are the nice way to communicate with the program or code we write. Comments are very useful to read and understand what's going on inside the program.

When you work large scale projects you often end up with write more lines of code. Let's say after few months or may few years you might want to see the code you wrote few months or years ago. In that situation comments will be very helpful to clearly communicate with you by saying what this code does or what why it was written.

In ruby, we can write comments in two ways. single line and multiline comments. Ruby single line comments


puts “today is  Friday”
puts “I am learning ruby”
Output
today is Friday I am learning ruby

Let's say you don’t want to print the second line which says “I am learning ruby”. You can do skip that in ruby using single line comments. 


puts “today is Friday”
# puts “I am learning ruby”

output:
today is Friday

Since we added a single line comment to the second line, when you execute the program it won’t be printed.

Now, let's move on to multiline comments.

If you want to skip more than one line or multiple lines in the code you can do that using ruby multiline comments. 


puts “Sunday”
puts “Monday”
puts “Tuesday”
=begin
puts “Wednesday”
puts “Thursday”
=end

When you run the above code ruby interpreter ignores the two lines we wrote inside the =begin and =end statement. In ruby, multi-line comments are written in between the =begin and =end blocks.

Tip: (Wrong Tick Mark)


= begin
 puts “This line won’t be printed”
= end
Don’t give any spaces between = and begin or end statement.

 (Right tick mark)
=begin
puts “This line won’t be printed”
=end

There is also another way to use ruby comments inside the code. Like explaining each line, you write about what is done.


name = “apple”       # this is the name of apple
number = 100         # the value 100 will be stored in the number memory location.

Remember this comments will not affect any code changes or output changes in the program. Let's say you want to print those two values.


puts name
puts number

Output
apple
100

Did you noticed, comments does not affect in our output!. This how ruby comments would work in rub programming language.
