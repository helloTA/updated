jQuery Installation




How to install jQuery

The jQuery installation main motto is to include the jQuery JavaScript file in the web page. We can do this in two ways to access the JavaScript file.

Download the jQuery library fromjQuery website and hosting it in web application

Downloading jQuery

There are two versions available for downloading the jQuery library:

Production version - this is for your live website because it has been minified and compressed Development version - this is for testing and development (uncompressed and readable code) Both versions can be downloaded from jQuery.

If you want to download the jQuery API, you have to place it on a web server. Once done, Your web pages start consuming jQuery library by specifying the directory path in <scirpt> tag as follows.

Note: the <script> tag should be inside the <head> section.

How to include jQuery in html file



    <!DOCTYPE html>
    <html>
    <head>
        <title>Including jQuery in HTML</title>
        <script type="text/javascript"  src="/jqfolder/1.7.1/jquery.min.js"></script>
    </head>
   <body>
         
Basic jQuery Example 
   </body>
   </html>
Referencing jQuery at Google CDN jQuery CDN :

If you don't want to download and make use of hosting jQuery in web pages, still there is an easiest way to include JavaScript file from a CDN (Content Delivery Network). Both Google and Microsoft has CDN to host them on to web pages.


To use jQuery from Google or Microsoft, use one of the following:

Google CDN: 

 <head>
  <script src="http://ajax.googleapis.com/ajax/libs/jquery/1.11.2/jquery.min.js"<</script>
</head>
Microsoft CDN: 


<head>
  <script src="http://ajax.aspnetcdn.com/ajax/jQuery/jquery-1.11.2.min.js"<</script>
</head>

Minimized vs. Full jQuery Versions

There are two versions of JQuery you can reference in your web pages. The minimized (minified), and the full version. There are no functional differences between these two versions. The only difference is the file size.


The full version contains the JavaScript code in an easy-to-read format. This makes reading and debugging the code easier. The disadvantage is that this file is much larger than the minimized version, resulting in more traffic and slower downloads.


The minimized version is a compacted edition of the full version. This version is very hard to read and debug inside. But the file is smaller, resulting in faster downloads, and less traffic. Usually, developers use the full version during development, and when the site is ready to go live, it will switch to the minimized edition.
