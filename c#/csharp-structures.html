C# Structures




What is Structure in C#

A structure is a collection of variables under a single name. These variables can be of different types, and each has a name which is used to select it from the structure. A structure is a convenient way of grouping several pieces of related information together.

Structures are used to represent a record. Suppose you want to keep track of your music album in a library. You might want to track the following attributes about each album:

  • Title
  • Director
  • Album Name
  • Album ID
  • Price
Syntax
[] struct <name>
{
  -- Define members
}

Code file: To define a structure under the project we are not provided with the structure item template in the add new item window like a class and an interface item templates. so, to define the project we use the code file item template which provides you a blank file with .cs extension so that we can define either a class or a structure or an interface also.

Defining a Structure

To define a structure, you must use the struct statement. The struct statement defines a new data type, with more than one member for your program.

For example, here is the way you can declare the Music Album structure:

  struct Album
{
   public string title;
   public string director;
   public string album_name;
   public int album_id;
   public double price
};
Example 
  struct MyStruct
   {
  int x;
  public MyStruct(int x)
  {
      this.x = x;
  }
  public void Display()
  {
      Console.WriteLine("Method under a structure :" + this.x);
  }

  static void Main()
  {
      MyStruct m1;
      m1.x = 10;
      m1.Display();
      MyStruct m2 = new MyStruct(20);
      m2.Display();

      Console.ReadLine();
  }
}

The above program prduces the following output

Output
Method under a structure :10
Method under a structure : 20
Structure Vs Class:
  1. The structure is actually a value type meaning memory allocated only on the stack. so there is no automatic memory management like class.
  2. Creating structure object is optional whereas in class is mandatory.
  3. Structure are used for representing the smaller volume of data whereas in class is for representing the larger volume of data.
  4. Zero argument constructor is optional in class, but it is mandatory in structure.
  5. The structure doesn’t support both implementations of interface and inheritance. whereas in class possible.
Consuming as a structure:

Even if a structure doesn’t support inheritance we can still consume a structure from the structures or classes by creating it an object.

To test this, try following program 

  class TestStruct
{
    static void Main()
{    
        MyStruct obj = new MyStruct(80);
        obj.Display();
        Console.ReadLine();
    }
}
Note:

C# language suffers from a criticism that it is not fully object oriented programming language because of this support that it gives for structure and this structure doesn’t support inheritance.

We can not escape from structure because all the predefined data types that come undervalue type category like int, float, bool etc are implemented as a structures in the base class libraries (BCL) within the system name space as int32, single, boolean etc.

Structure Example in C#

using System;
struct Albums
{
   public string title;
   public string director;
   public string album_name;
   public int album_id;
};  

public class exampleStructure
{
   public static void Main(string[] args)
   {

      Albums Album1;   /* Declare Album1 of type Album */
      Albums Album2;   /* Declare Album2 of type Album */

      /* album 1 specification */
      Album1.title = "Mama Africa";
      Album1.director = "Akon"; 
      Album1.album_name = "Mama Africa";
      Album1.album_id = 407;

      /* album 2 specification */
      Album2.title = "Sorry, Blame It on Me";
      Album2.director = "Akon";
      Album2.album_name =  "Sorry, Blame It on Me";
      Album2.album_id = 700;

      /* print Album1 info */
      Console.WriteLine( "Album 1 title : {0}", Album1.title);
      Console.WriteLine("Album 1 director : {0}", Album1.director);
      Console.WriteLine("Album 1 album_name : {0}", Album1.album_name);
      Console.WriteLine("Album 1 album_id :{0}", Album1.album_id);

         /* print Album2 info */
      Console.WriteLine( "Album 2 title : {0}", Album2.title);
      Console.WriteLine("Album 2 director : {0}", Album2.director);
      Console.WriteLine("Album 2 album_name : {0}", Album2.album_name);
      Console.WriteLine("Album 2 album_id :{0}", Album2.album_id);


      Console.ReadKey();

   }
}

The above program will produce the following output

Album 1 title : Mama Africa
Album 1 director : Akon
Album 1 album_name : Mama Africa
Album 1 album_id :407
Album 2 title : Sorry, Blame It on Me
Album 2 director : Akon
Album 2 album_name : Sorry, Blame It on Me
Album 2 album_id :700
