C# Exception Handling




What is Exception Handling

Exception handling is the process of responding to the occurrence, during computation, of exceptions – anomalous or exceptional conditions requiring special processing – often changing the normal flow of program execution. It is provided by specialized programming language constructs or computer hardware mechanisms.

The exception is an error which occurs in the program execution. Whenever we write a program there are chances of coming across 2 different types of errors in the program.

  1. Compile-time errors
  2. Runtime errors

An error which occurs in the program due to a syntactical mistake is a “Compile-time errors” and these errors are not at all dangerous. so we don’t require to take them into consideration at all.

An error which occurs in the program while the program execution is going on is known as “Runtime errors” and these errors due to various reasons like wrong implementation of logic missing of required resources, wrong input supply to the program etc.

Runtime errors are very dangerous because whenever a runtime error occurs in the program, the program terminates abnormally on the same line without executing any other lines of code in the program.

Exceptions are introduced and responsible for abnormal termination of programs when runtime errors occur in the program.

What are Exceptions?

An exception is a class which is implemented under base class libraries(BCL). Representing each and every different type of runtime error one exception class has been defined under BCL like

  1. IndexOutOfRangeException,
  2. NullReferenceException,
  3. FormatException,
  4. DividedByZeroException.

When a runtime error occurs in a program the object of an appropriate exception class comes into the picture and first terminates the program abnormally on the line and displays an error message describing the error that got occurred.

Exception handling:

As discussed above whenever a runtime error occurs our program will be terminating abnormally. If we wanted to stop this abnormal termination we need to go for handling exceptions. so that we will be getting the following benefits.

  1. We can perform a corrective action to resolve the problem that got occurred because of the error Ex: roll backing the transaction ( money)
  2. We can display a user-friendly error message so that users can understand the error and also resolve it if possible. Ex: message will be given to user if user entered the wrong destination id
  3. As the abnormal termination is stopping over we can continue with the approach and executes the stat’s that are no way related with the error.

Syntax

try
{
   //  statements which will be causing runtime errors
   //  statements which don't require execution when the runtime error occurs.
}
catch(&l;exception> <var>)
{
   //  statements which require execution only when a runtime error occurs.
}
--- < multiple catch blocks if required >-----
finally{
 //statements to be executed even runtime error occurs
}

Example

class ExceptionDemo
{
static void Main()
{
    int x, y, z;

    try
    {
        Console.WriteLine("enter x value:");
        x = int.Parse(Console.ReadLine());
        Console.WriteLine("enter y value:");
        y = int.Parse(Console.ReadLine());

        z = x / y;
        Console.WriteLine(z);
    }
    catch (DivideByZeroException ex)
    {
        Console.WriteLine("result :Divided value can't be zero ");
    }
    catch (FormatException ex)
    {
        Console.WriteLine("result : Input must be numeric");
    }
    catch (Exception ex)
    {
        Console.WriteLine(ex.Message);
    }
    finally
    {
        Console.WriteLine("End of the program and finally block gets executed");
    }

    Console.ReadLine();
}
}

If you run the above example code, it will produce the following output

Output
enter x value : 2
enter y value: 0
result : Divided value can't be zero
End of the program and finally block gets executed
enter x value : g
result: Input must be numeric
End of the program and finally block gets executed

We can use these 3 blocks in 3 different combinations

try & catch: in this case, if any runtime error occurs in the program exception will be caught stopping the abnormal termination.


try, catch & finally: in this case the behavior will be same as above and along with it the statements under finally block gets executed at any cost.

try & finally: In this case, if any runtime error occurs in the program abnormal termination will occur because there is no catch block to handle the exception but the statements under finally block will be still executed.

How to raise an exception?

Exceptions are of 2 types

  1. System exceptions
  2. Application exceptions

If an exception occurs in a program implicitly we call it as system exception where the implicit exception occurs basing on predefined conditions.

example DividedByZeroException, FormatException etc

If an exception occurs in a program which is explicitly raised by a program we call it as Application Exceptions

If user wants to raise an exception in thr program they need to follow the below process

step 1: create the object of an exception class

step 2: throw the object i.e., created with the help of throw statement.>

ex: FormatException ex = new FormatException();
    throw ex;

Example

 class DividedByOddException : Exception
{
    public override string Message
    {
        get
        {
            return "attempted to divide by the odd number";
        }
    }
}

class ThrowDemo
{
    static void Main()
    {
        int x, y, z;
        Console.WriteLine("enter x value:");
        x = int.Parse(Console.ReadLine());
        Console.WriteLine("enter y value:");
        y = int.Parse(Console.ReadLine());

        if (y % 2 > 0)
            throw new DividedByOddException();
        z = x / y;
        Console.WriteLine(z);

        Console.WriteLine("End of the program");
        Console.ReadLine();
        
    }
}
