C# Comments




What is a comment in C#

In computer programming, a comment is a programming language construct used to embed programmer-readable annotations in the source code of a computer program. Those annotations are potentially significant to programmers but are generally ignored by compilers and interpreters.

Comments can be used to document what the program does and what specific blocks or lines of code do. C# compiler doesn’t compile comment code so we can include the comments any where in the program.


Comments are very useful for when we write programming language for better understanding in the days to come.

Comments are of 2 types in C#

Single line comments

Multi-line comments


Single line comments

Single comments are written inside the program using two backslashes(//) to explain about the specific line of the program


// this is single line comment
Example of single line comments: 

 public void Add()
    {
      int resultant ; // resultant value will be captured after adding number1 and number2
    resultant = number1 + number2;
    Console.WriteLine(“Addition of number1 and number2 is “+resultant);
         }

In this example,single line comment(//) explaining about what the purpose of resultant variable.

Multi line comments

Often when you code, you may want to comment out an entire block of code statements. Under normal circumstances, to do so you must Comment out each line individually. Instead of make use of single line commenting for each and every line, simply use multi line commenting.

/* this is a multi-line comment
I can write any no of words inside this.
All comments are ignored by the interpreter.
*/
Example of multi-line comments in C#: 

   /* const int level = 100;
const decimal turnaround = 45.67; */

Like the above example, we can comment n number of lines.

Use comments only for portions of code that are difficult to understand.

As we know C# is an object-oriented programming language and In Object-Oriented Programming basically we come across a concept called Object.

What is an Object?
An object is basically a block of memory that has been allocated and configured according to the blueprint.

A program may create many objects of the same class. Objects are also called instances, and they can be stored in either a named variable or in an array or collection.

Let's try to understand Object usage with console application example


using System
namespace AdditionSample
{
    class Addition
    {
        // member variables
        int number1;
        int number2;
        public void IntializeData()
        {
            number1= 10;    
            number2= 6;
        }
        public void Add()
        {
            int resultant;
resultant = number1 + number2;
Console.WriteLine(“Addition of number1 and number2 is “+resultant);
        }
    }
    
    class ProcessTask
    {
        static void Main(string[] args)
        {
            Addition add = new Addition();
            add.IntializeData();
            add.Add();
            Console.ReadLine();
        }
    }
}

Output
The addition of number1 and number2 is 16.

From this example, we have 2 classes(Addition and ProcessTask) declared under AdditionSample namespace where Addition class is to perform the addition mathematical operation and ProcessTask is to starting the execution of the program.


In Order to perform an addition operation from this example Object plays important role i.e., By using the class object we can access members(methods declared in Addition class) of it and Object creation can be done by using “new” keyword.
