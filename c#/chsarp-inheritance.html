C# Inheritance




What is Inheritance

In object-oriented programming (OOP), inheritance is when an object or class is based on another object or class, using the same implementation (inheriting from a class) specifying implementation to maintain the same behavior (realizing an interface; inheriting behavior).s

Acquiring the members of the class into one or more new classes is known as the inheritance which can be done by establishing the parent - child relationship between the classes.

It’s an approach which provides reusability of code i.e., the members that are defined under a class can be consumed from other classes by the parent- child relationship between the classes. So that the members of parent class can now be consumed under its child classes.

In inheritance, we call parent class as base class and child class as derived class.


Syntax for C# Inheritance 
 [<modifiers>] class < classname> : <parent class name>
Example 
class ParentClass
{

}
class ChildClass : ParentClass
{
    consume all members of parent class
}
Note:

Inheritance private members of a class can not be acquired by it’s child class.

Example program for C# Inheritance

 class ClassA
{
    public void Add(int x, int y)
    {
        Console.WriteLine(“Addition of x and y is : ” + (x + y));
    }
}

class ClassB : ClassA
{
    public void Multiply(int x, int y)
    {
        Console.WriteLine(“Multiplication of x and y is : ” + (x * y));
    }
    static void Main()
    {
        ClassB b = new ClassB();
        b.Add(23, 90);
   b.Multiply(3, 9);

        Console.ReadLine();
    }
}

The above code will produce the following output

Addition of x and y is : 113
Multiplication of x and y is : 27

As we see the inheritance usage from the example program ClassB can consume the members of ClassA method ‘Add’.

Single Inheritance


Multiple Inheritance


If a class has one immediate parent class to it we called as single inheritance and if a class has more than one immediate parent classes we call it as multiple inheritance.

Initializing Base Class

The derived class inherits the base class member variables and member methods. Therefore the super class object should be created before the subclass is created. You can give instructions for superclass initialization in the member initialization list.

The following program demonstrates this: 

using System;
namespace RectangleApplication
{
   class Rectangle
   {
      //member variables
      protected double length;
      protected double width;
      public Rectangle(double l, double w)
      {
         length = l;
         width = w;
      }
      
      public double GetArea()
      {
         return length * width;
      }
      
      public void Display()
      {
         Console.WriteLine("Length: {0}", length);
         Console.WriteLine("Width: {0}", width);
         Console.WriteLine("Area: {0}", GetArea());
      }
   }//end class Rectangle  
   
   class Tabletop : Rectangle
   {
      private double cost;
      public Tabletop(double l, double w) : base(l, w)
      { }
      public double GetCost()
      {
         double cost;
         cost = GetArea() * 70;
         return cost;
      }
      public void Display()
      {
         base.Display();
         Console.WriteLine("Cost: {0}", GetCost());
      }
   }
   class ExecuteRectangle
   {
      static void Main(string[] args)
      {
         Tabletop t = new Tabletop(4.5, 7.5);
         t.Display();
         Console.ReadLine();
      }
   }
}

If you run the above program it will produce the folloqing output

Output
Length: 4.5
Width: 7.5
Area: 33.75
Cost: 2362.5
