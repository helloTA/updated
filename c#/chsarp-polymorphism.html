C# Polymorphism




What is Polymorphism

In object-oriented programming, polymorphism refers to a programming language's ability to process objects differently depending on their data type or class. More specifically, it is the ability to redefine methods for derived classes.

Behaving in different ways depending up on input received is known as polymorphism i.e., whenever the input changes automatically output/behavior also changes.

There are different approaches to implementing the polymorphism in c# language.

Method Overloading

Operator Overloading


It’s an approach of defining multiple methods in the class with same name by changing their signature.Changing the signature in the sense we can either change either no.of parameters being passed to the method or type of parameters being passed to the method or order of parameters being passed to the method.

Example

public void show()
public void show(int x)
public void show(string x)
public void show(int x, string y)
public void show(string x, int y)
Note:

A change in the return type of a method will not be taken into consideration in overloading. public string show() // invalid

Example

This example program will explain more about the polymorphism

  class LoadDemo
  {
    public void Show(int a)
    {
        Console.WriteLine("Only one variable to show here :  "+ a);
    }
    public void Show(int a, int b)
    {
        Console.WriteLine("Add result is:" + (a+b));
    }
    public void Show(int a, string b)
    {
        Console.WriteLine("Concatenate int with string result is : " + a +“ age, ”+ b);
    }
    static void Main()
    {
        LoadDemo obj = new LoadDemo();

        obj.Show(34);
        obj.Show(12, 45);
        obj.Show(23,"Hi M.r Been");


        Console.ReadLine();
    }
  }

When you run the above program it will produce the following output

Only one variable to show here : "+ a
Add result is: 57
Concatenate int with string result is :23 age , Hi M.r Been

Method Overloading

You can have multiple definitions for the same method name in the same scope. The definition of the method must differ from each other by the types and/or the number of arguments in the argument list. You cannot overload function declarations that differ only by return type. The following example shows using method print() to print different data types:

Example
using System;
namespace PolymorphismApplication
{
   class Printdata
   {
      void print(int i)
      {
         Console.WriteLine(" Integer value: {0}", i );
      }

      void print(double f)
      {
         Console.WriteLine(" Floating point value: {0}" , f);
      }

      void print(string s)
      {
         Console.WriteLine("String value :  {0}", s);
      }
      static void Main(string[] args)
      {
         Printdata p = new Printdata();
         
         // Call print to print integer
         p.print(100);
         // Call print to print float
         p.print(100.12);
         
         // Call print to print string
         p.print("Welcome to Tutorials Academy");
         Console.ReadKey();
      }
   }
}

If you run the above code, it will produce the following output

Integer value: 100
Floating point value: 100.12
String value : Welcome to Tutorials Academy
