Java Modifiers




Java Modifiers


Modifers are a type of keywords in java which allows you to change the behavior of your program from accessing the data inside the program. You can also say controling the flow of your data in the program.

There are two types of modifiers in java.

  • Java Access Modifiers
  • Non Access Modifiers

The access modifiers in java specify how you can access a data member or whether you can actually have the access to it or not?. It not only limited to data member, it also includes methods, constructors and classes.


How can you use the java modifiers?

Using the java modifiers is easy, you just need to mention the keyword of the modifier you want to you. There are 3 different types of java modifiers you can use those are called public, private and protected

The following example demonistrates the usage of java public modifer, java private modifier, and java protected modifier.

public class yourclassName {
   // ...
}
private int yourvariableName;
static final float amount = 6500.65;
protected static final int MAXVALUE = 42000;
public static void main(String[] arguments) {
   // You can write the body or logic of your code here
}

Before you move, you need to understand few points about java access control modifiers

The good part of java is, it provides set of access modifiers to control the access levels of our program, for example classes, variables, methods etc...

However there are four types of access levels which you need to understand.

When you use the java public access modifier, then the whole program or data will be visible to the all(which means public).

When you use private as your java access modifier .i.e java private modifier, the data or program methods are visible to the class only.

When you use protected as your java access modifier .i.e java protected modifier then the program data and methods are visible or accessible to the package and all subclasses.


Java Non Access Modifiers:

Java provides a number of non-access modifiers to achieve many other functionality.

  • The static modifier for creating class methods and variables
  • The final modifier for finalizing the implementations of classes, methods, and variables.
  • The abstract modifier for creating abstract classes and methods.
  • The synchronized and volatile modifiers, which are used for threads.
