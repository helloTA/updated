Java Objects and Classes




Java is a true Object Oriented language and, therefore, the underlying structure of all Java programs is classes.

Object: Anything we wish to represent in Java must be encapsulated in a class that defines the “state” and “behaviour” of the basic program components known as objects.

Classes create objects and objects use methods to communicate between them. They provide a convenient method for packaging a group of logically related data items and functions that work on them.

Objects in Java

In Object oeriented programming, an object is referred to an entity which has some behavior and state. The behavior often changes based on the state. For example a Car is an object, it has a behavior like move left, move right, and state like idle(which means it is in parking state) and moving state(which means the car is currently running or active state).

Every Object in Java has three characteristics:

state: represents data (value) of an object.

behavior: represents the behavior (functionality) of an object such as move left, move right etc.

identity: Object identity is typically implemented via a unique ID. The value of the ID is not visible to the external user. But,it is used internally by the JVM to identify each object uniquely. For example car Id

Syntax for creaing objects in java 

  Foo ref = new Foo();

Creating an object is not limted to one way in java.

What are all the different ways to create an object in Java?

There are four different ways to create objects in java:

  1. Using new keyword
  2. Using Class.forName()
  3. Using clone()
  4. Using object deserialization

Let us understaind the above four ways more clearly.

Using new keyword

This is the most common way to create an object in java, almost 99% of objects are created in this way.

Syntax 
MyObject object = new MyObject();

Using Class.forName()

If we know the name of the class & if it has a public default constructor we can create an object in this way.

MyObject object = (MyObject) Class.forName("subin.rnd.MyObject").newInstance();

Using clone() The clone()

It can be used to create a copy of an existing object.

MyObject anotherObject = new MyObject(); MyObject object = anotherObject.clone();

Using object deserialization

Object deserialization is nothing but creating an object from its serialized form.

ObjectInputStream inStream = new ObjectInputStream(anInputStream ); MyObject object = (MyObject) inStream.readObject();

Now you know how to create an object. But its advised to create objects only when it is necessary to do so.



Example program to create an object in java

  public class Laptop{

   public Laptop(String name){
      // This constructor has one parameter, called Laptop's name.
      System.out.println("The name of my laptop is :" + name ); 
   }
   public static void main(String []args){
      // this below statement would create an object myLaptop
      Laptop myLaptop = new Laptop( "MacBook Air" );
   }
}

The above program creates an object called myLaptop.

Output 
  The name of my laptop is: MacBook Air


Classes in java

Class

In an object-oriented programming language, a program is the collection of members like variables and functions. But those members are enclosed under a special container known as a “class”. Which gives the basic security for contents that are present inside and we call the class as the wrapper.

A class is a user-defined data type very much same as a structure in our procedural programming languages.

As a class is a data type to consume the class or call the members of class we must create an object.

We can not consume a data type directly whether it is predefined or less defined. So, to consume a datatype first we need to create a copy of the type.

Example

int = 100;             // invalid
int x = 100;          //valid

In the above case, x is a copy of type ‘int’ and the memory allocation is done for that x only. For the data type, there is no allocation of memory.

Exactly in the same way if we define a class or a structure here also datatypes only but user defined. So, to consume them also we need to create a copy of types.

Copies of simple types like int, float, char etc are known as variables whereas copies of complex types like user defined structures or class will be called as “object”.

Defining a Class in Java

The general syntax for defining a class in Java is shown below. 

class MyClassName{
  . . .
} //End of class definition.

This syntax defines a class and creates a new type named MyClassName. The definitions of variables and methods are inserted between the opening and closing braces.

Syntax to define a class in Java: 

<modifier> class <class_name>{  
    data member;  
    method;  
}
